    - 疑问
        - 那agent怎么知道自己要使用那个mcp  
            Agent 本身并不知道有哪些 MCP，也不应该把所有 MCP 写死在代码里。正确做法是通过 Tool Registry（工具注册中心）+ MCP 元数据描述 + LLM Tool Calling 来实现动态选择，按需注入 而不是全部注入企业级方案：Tool RAG 把所有工具向量化。
        
        - tool和mcp的关系  
            tool是让大模型知道这个是用来干嘛的有描述 mcp更接近实际的实现 controller和server层的区别

        -  engine和tool的边界
			1. 单一职责  
			2. 无流程编排  
			3. 输入 -> 输出  
			4. 不关心上下文业务  
			5. 不知道“为什么调用它”
	        engine是工作流编排注重流程编排
	          组合多个 Tool  
				1. 有业务流程  
				2. 有状态（state）  
				3. 有顺序 / 分支 / 循环  
				4. 决定“什么时候用Tool”
	        tool是能力做这个功能
	        mcp是去哪里执行这个执行功能
